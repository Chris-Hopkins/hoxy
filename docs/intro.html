Hoxy: Introduction

Hoxy is an HTTP hacking tool for JavaScript programmers. It operates in the same general space as Charles or Fiddler, but rather than having a GUI, it exposes a programming API for tampering with HTTP traffic. A few key features:

  • Manipulate request and response bodies as jQuery or JSON objects.
  • Inspect and manipulate URLs, headers, status codes, etc.
  • Support for direct and reverse proxying over both HTTP and HTTPS.
  • Simulate latency and low-bandwidth connections.
  • Overlay local filesystems over remote ones.

With a little imagination, the possibilities are limitless. But to get you started, here a few ideas for things you could do.

  • Replace your Facebook frineds with pictures of unicorns.
  • Add bizarre HTTP request headers to confuse server admins.
  • Strip ads from web pages for personal browsing.
  • Fuzz test your site's load sequence by adding random amounts of latency.
  • Test JavaScript changes in production, without pushing code to production.

These docs are for Hoxy 3.x. See the incantations section for code samples.

How it works

Hoxy functions as a normal proxy standing between the client and server. You intercept traffic during either the request or response phase, or both. The phase lifecycle section contains a more complete description, but here's a simplified illustration. 

  time ==>
  -----------------------
  server:       3
  -------------/-\-------
  hoxy:       2   4
  -----------/-----\-----
  client:   1       5
  1. Client sends request.
  2. Hoxy intercepts the request.
  3. Server receives request and sends response.
  4. Hoxy intercepts the response.
  5. Client receives response.
